## "SlipStream GUI is damaged" error

This is the most common macOS issue, caused by Gatekeeper security settings blocking unsigned applications.

<AccordionGroup>
  <Accordion title="Option 1: Remove quarantine attribute (Recommended)">
    This is the fastest and most reliable method.

    1. Open **Terminal** (Applications → Utilities → Terminal)
    2. Run this command:
       ```bash
       xattr -cr /Applications/SlipStream\ GUI.app
       ```
    3. Press Enter and try opening the app again

    <Note>This command removes the quarantine flag that macOS adds to downloaded applications.</Note>
  </Accordion>

  <Accordion title="Option 2: Allow in System Settings">
    Let macOS know you trust the application.

    1. Try to open SlipStream GUI (it will be blocked)
    2. Go to **System Settings** → **Privacy & Security**
    3. Scroll down to the **Security** section
    4. You should see a message: _"SlipStream GUI" was blocked from use because it is not from an identified developer_
    5. Click **"Open Anyway"**
    6. Confirm by clicking **"Open"** in the dialog

    <Note>You only need to do this once. macOS will remember your choice.</Note>
  </Accordion>

  <Accordion title="Option 3: Right-click to open">
    Bypass Gatekeeper for the first launch.

    1. Locate SlipStream GUI in Applications
    2. **Right-click** (or Control-click) on the app
    3. Select **"Open"** from the context menu
    4. Click **"Open"** in the confirmation dialog

    After the first successful launch, you can open the app normally.
  </Accordion>
</AccordionGroup>

<Warning>
If all three options fail, the DMG file may be corrupted. Download a fresh copy from the [releases page](https://github.com/mirzaaghazadeh/SlipStreamGUI/releases/latest).
</Warning>

## Binary execution permissions

The SlipStream client binary must be executable. The app handles this automatically, but you may need to verify manually if issues persist.

<AccordionGroup>
  <Accordion title="Check binary permissions">
    **For Apple Silicon (M1/M2/M3):**
    ```bash
    ls -l /Applications/SlipStream\ GUI.app/Contents/Resources/binaries/slipstream-client-mac-arm64
    ```

    **For Intel Macs:**
    ```bash
    ls -l /Applications/SlipStream\ GUI.app/Contents/Resources/binaries/slipstream-client-mac-intel
    ```

    The output should show `-rwxr-xr-x` (executable permissions).
  </Accordion>

  <Accordion title="Manually set execute permissions">
    If permissions are missing:

    **Apple Silicon:**
    ```bash
    chmod +x /Applications/SlipStream\ GUI.app/Contents/Resources/binaries/slipstream-client-mac-arm64
    ```

    **Intel:**
    ```bash
    chmod +x /Applications/SlipStream\ GUI.app/Contents/Resources/binaries/slipstream-client-mac-intel
    ```

    <Note>The app automatically sets these permissions on startup, but manual intervention may be needed if the app bundle was extracted incorrectly.</Note>
  </Accordion>

  <Accordion title="Architecture mismatch">
    If you're running the wrong binary for your Mac:

    **Check your Mac's architecture:**
    ```bash
    uname -m
    ```
    - `arm64` = Apple Silicon (M1/M2/M3)
    - `x86_64` = Intel

    **Download the correct version:**
    - [Apple Silicon DMG](https://github.com/mirzaaghazadeh/SlipStreamGUI/releases/latest/download/SlipStream-GUI-macOS-ARM64.dmg)
    - [Intel DMG](https://github.com/mirzaaghazadeh/SlipStreamGUI/releases/latest/download/SlipStream-GUI-macOS-Intel.dmg)

    <Note>Apple Silicon Macs can run Intel binaries through Rosetta 2, but native ARM64 binaries perform better.</Note>
  </Accordion>
</AccordionGroup>

## System proxy configuration

On macOS, system proxy configuration uses `networksetup` and may require administrator privileges.

<AccordionGroup>
  <Accordion title="Administrator password prompts">
    When you enable "Configure System Proxy", macOS will prompt for your administrator password.

    **Why this happens:**
    - `networksetup` requires admin privileges to modify network settings
    - This is a macOS security feature
    - The app does not store or have access to your password

    **If the prompt doesn't appear:**
    - The app will log an error
    - Configure proxy manually in System Settings
  </Accordion>

  <Accordion title="Manual proxy configuration">
    If automatic configuration fails:

    1. Open **System Settings** → **Network**
    2. Select your active network (Wi-Fi or Ethernet)
    3. Click **Details**
    4. Go to the **Proxies** tab
    5. Enable **Web Proxy (HTTP)** and **Secure Web Proxy (HTTPS)**
    6. Set **Web Proxy Server**: `127.0.0.1:8080`
    7. Set **Secure Web Proxy Server**: `127.0.0.1:8080`
    8. Click **OK**
  </Accordion>

  <Accordion title="Network interface detection">
    The app automatically detects and configures the active network interface.

    **Preferred interfaces (in order):**
    1. Wi-Fi
    2. Ethernet
    3. USB 10/100/1000 LAN
    4. Thunderbolt Bridge

    **Check configured interface:**
    ```bash
    networksetup -listallnetworkservices
    ```

    **Manually configure a specific interface:**
    ```bash
    # Replace "Wi-Fi" with your interface name
    networksetup -setwebproxy "Wi-Fi" 127.0.0.1 8080
    networksetup -setsecurewebproxy "Wi-Fi" 127.0.0.1 8080
    networksetup -setwebproxystate "Wi-Fi" on
    networksetup -setsecurewebproxystate "Wi-Fi" on
    ```
  </Accordion>

  <Accordion title="Proxy persists after quitting app">
    If the system proxy remains configured after stopping the VPN:

    **Disable proxy manually:**
    ```bash
    # Replace "Wi-Fi" with your interface name
    networksetup -setwebproxystate "Wi-Fi" off
    networksetup -setsecurewebproxystate "Wi-Fi" off
    ```

    **Check current proxy status:**
    ```bash
    networksetup -getwebproxy "Wi-Fi"
    networksetup -getsecurewebproxy "Wi-Fi"
    ```

    <Note>The app only disables proxy configurations it created (pointing to 127.0.0.1:8080).</Note>
  </Accordion>
</AccordionGroup>

## macOS-specific logs

Access detailed logs for debugging:

<AccordionGroup>
  <Accordion title="Application logs">
    **View console logs:**
    1. Open **Console.app** (Applications → Utilities → Console)
    2. Select your Mac in the sidebar
    3. Filter for "SlipStream GUI"

    **Common log locations:**
    - Electron logs: `~/Library/Logs/SlipStream GUI/`
    - Settings file: `~/Library/Application Support/SlipStream GUI/settings.json`
  </Accordion>

  <Accordion title="Network configuration logs">
    **Check networksetup output:**
    ```bash
    # View all network services
    networksetup -listallnetworkservices

    # Check proxy configuration
    networksetup -getwebproxy "Wi-Fi"
    networksetup -getsecurewebproxy "Wi-Fi"

    # Check proxy bypass domains
    networksetup -getproxybypassdomains "Wi-Fi"
    ```
  </Accordion>

  <Accordion title="Port usage on macOS">
    **Check if ports are in use:**
    ```bash
    sudo lsof -i :8080
    sudo lsof -i :5201
    ```

    **Kill processes using ports:**
    ```bash
    sudo lsof -ti:8080 | xargs kill -9
    sudo lsof -ti:5201 | xargs kill -9
    ```
  </Accordion>
</AccordionGroup>

## Sharing VPN over Wi-Fi

Share your VPN connection with iOS devices on the same network.

<AccordionGroup>
  <Accordion title="Find your Mac's IP address">
    **Option 1: Terminal**
    ```bash
    ifconfig | grep "inet " | grep -v 127.0.0.1
    ```

    **Option 2: System Settings**
    1. Open **System Settings** → **Network**
    2. Select your Wi-Fi connection
    3. Your IP address is shown (e.g., `192.168.1.100`)
  </Accordion>

  <Accordion title="Configure iOS device">
    1. On iPhone/iPad: **Settings** → **Wi-Fi**
    2. Tap **(i)** next to your network
    3. Scroll to **HTTP Proxy** → **Manual**
    4. **Server**: Your Mac's IP (e.g., `192.168.1.100`)
    5. **Port**: `8080`
    6. Tap **Save**
  </Accordion>

  <Accordion title="Firewall configuration">
    If iOS can't connect, check your Mac's firewall:

    1. **System Settings** → **Network** → **Firewall**
    2. Ensure firewall allows incoming connections on port 8080
    3. Or temporarily disable firewall to test

    **Allow port 8080 via Terminal:**
    ```bash
    sudo /usr/libexec/ApplicationFirewall/socketfilterfw --add /Applications/SlipStream\ GUI.app
    sudo /usr/libexec/ApplicationFirewall/socketfilterfw --unblockapp /Applications/SlipStream\ GUI.app
    ```
  </Accordion>
</AccordionGroup>

## Other macOS issues

<AccordionGroup>
  <Accordion title="App doesn't appear in Dock">
    The app should appear in the Dock when running. If it doesn't:

    1. Check Activity Monitor for "SlipStream GUI" process
    2. Try force-quitting and reopening
    3. Reinstall the application
  </Accordion>

  <Accordion title="DMG won't mount">
    If the DMG file won't open:

    1. Download the DMG again (may be corrupted)
    2. Try mounting manually:
       ```bash
       hdiutil attach ~/Downloads/SlipStream-GUI-macOS-*.dmg
       ```
    3. Check disk permissions:
       ```bash
       diskutil verifyVolume /
       ```
  </Accordion>

  <Accordion title="App crashes on launch">
    If the app crashes immediately:

    1. Check Console.app for crash logs
    2. Reset settings:
       ```bash
       rm ~/Library/Application\ Support/SlipStream\ GUI/settings.json
       ```
    3. Reinstall from a fresh download
  </Accordion>
</AccordionGroup>
